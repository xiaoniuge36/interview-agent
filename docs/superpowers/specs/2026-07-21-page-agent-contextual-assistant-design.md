# 前后台 Page Agent 场景化助手设计说明

## 目标

让现有前台和后台 Page Agent 理解用户当前所在页面与工作阶段，使首屏快捷入口、抽屉提示和模型运行时指令都切换为当前最相关的安全行动。功能继续集成在现有用户端与后台管理端，不创建独立系统，不改变会话、模型配置、权限或持久化结构。

## 范围与边界

- 前台关注求职者的刷题计划、练习中、复盘、个人档案、JD 与模型配置。
- 后台关注看板、资料导入、审核工作台、运行观测、模型治理与账号管理。
- 场景只由当前路由和页面角色推导；不额外请求业务数据、不引入主动提醒或全局轮询。
- Agent 仅提供只读查询、解释和现有页面导航。创建练习、提交答案、AI 评估、生成整轮复盘、审核、发布、导出、账号及模型修改仍须由用户在既有页面明确确认。
- 不向模型提供 API Key、Token、密码、手机号、邮箱或完整页面 DOM。

## 方案比较

1. 固定快捷入口：实现最少，但与用户当前任务脱节。
2. 路由场景化助手：根据用户正在处理的页面换出相关行动，成本低、可预测且不增加数据风险。采用此方案。
3. 主动运营提醒：需要持续采集指标、定义打扰策略和更复杂的告警治理，不属于本次范围。

## 前台体验

前台的场景规则由当前 pathname 推导，默认退回到全局训练教练场景。

- 首页和题库：展示今日训练安排、薄弱项强化、最近失分与进入复盘。
- 练习空间：展示开始前的题单选择、推荐依据解读、练习中解题思路与保存前检查、练习完成后的复盘入口。不会替用户开始、保存、提交或生成复盘。
- 复盘中心：展示报告解读、薄弱项归纳、下一轮训练方向与继续练习入口。
- 个人档案、JD 或设置：展示完善画像、检查 JD 完整性、连接可用模型等入口；没有模型时优先引导到设置。

抽屉首屏显示场景标题、简短说明和场景化快捷卡；对话运行时指令同时获得场景标识、允许的只读能力和禁止的写操作，避免模型把通用建议误当作当前页面动作。

## 后台体验

后台的场景规则由当前 hash 视图和管理员角色推导，默认退回到运营总览。

- 治理总览与数据看板：展示指标解读、异常趋势定位、AI 用量查询（仅平台管理员）和对应页面导航。
- 资料导入：展示待处理批次、导入进度解释与进入审核工作台；不启动导入或删除批次。
- 审核工作台：展示待审候选题、来源批次定位、审核规则说明；不审核、发布或批量处理。
- 运行观测：展示失败/降级运行、追踪线索和模型消耗分析；不重试、终止或更改运行。
- 模型治理与账号管理：展示连接健康、模型用量、账号角色说明和人工治理入口；不显示凭证，不修改模型或账号。

快捷动作继续使用现有安全的只读工具和导航工具，按权限隐藏平台管理员专属 AI 用量入口。运行时提示明确当前管理场景和允许的处理范围。

## 代码结构

- 前后台各增加一个纯函数场景模块：输入 `pathname` 或 `hash` 与角色，输出场景 id、标题、说明、快捷动作和运行时上下文。
- 现有快捷动作文件变为通用默认动作；抽屉在空会话时优先渲染当前场景动作。
- Widget 在浏览器端读取当前路由并将场景上下文传递给 Drawer 与 runtime；不改变 conversation hook、API 或数据库。
- runtime 在既有安全系统指令后追加简短场景上下文，使同一 Agent 能在会话恢复后理解用户当前页面。

## 错误处理与可访问性

- 未识别路由使用默认场景，始终保留可用的通用快捷入口。
- 场景动作的 prompt 不包含敏感信息和隐式写入指令。
- 快捷卡保留按钮语义、可见标题与禁用状态；运行中、未配置模型、会话加载失败的既有反馈保持不变。

## 验收与测试

1. 前台和后台路由都能映射到稳定、可读的场景；未知路由退回默认场景。
2. 各场景展示的快捷入口与该页面任务匹配，且不包含自动审核、发布、导出、创建练习、提交答案或消耗模型的指令。
3. 前台练习与复盘、后台导入与审核、后台运行观测至少各覆盖一个定向测试。
4. 运行时指令包含当前场景且仍保留既有安全约束。
5. 通过相应前端单测、lint、typecheck、build，并通过本地运行页面的关键交互检查。
