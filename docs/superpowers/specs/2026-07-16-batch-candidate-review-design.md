# 候选题按来源批量审核设计

## 目标

将管理端候选题审核从单题抽屉内的状态下拉框，升级为可按同一导入资料批量处理的审核流。审核人员必须能在列表和单题详情中识别候选题的来源导入文件。

## 范围

- 候选题列表返回并展示来源导入任务的 ID 与标题；没有导入来源的题目明确标为“非导入来源”。
- 审核列表支持多选，同一次批量操作只允许处理同一来源任务下的候选题。
- 批量操作支持“通过”“需修改”“驳回”，可统一写入审核备注。
- 单题审核抽屉显示来源资料入口，并使用明确的状态按钮取代下拉框。
- 发布到正式题库继续保持单题操作，不纳入批量操作。

## 不在范围内

- 不调整导入候选题生成逻辑。
- 不提供批量发布题库。
- 不改变已有单题编辑字段、权限模型或审核状态枚举。

## 数据与接口

### 列表来源信息

扩展 `CandidateReview` 返回值，新增可空来源对象：`{ id, title }`。Product API 查询候选题时关联 `ImportTask` 并映射此对象；CSV 导出维持现有导入任务 ID 字段。

### 批量审核

新增受 `candidate:review` 权限保护的批量审核接口。请求包含：

- `candidateIds`：1 至合理上限且不重复的候选题 ID 列表；
- `status`：`approved`、`needs_edit` 或 `rejected`；
- `reviewNotes`：可空的统一审核备注。

服务端在同一事务中加载所有候选题，并校验：全部属于当前租户、尚未发布、且 `importTaskId` 完全一致。校验失败时不写入任何记录；成功后逐题更新状态、备注与 revision，并为每题写入既有 `candidate:review` 审计记录。

## 前端交互

### 审核列表

- 新增“来源资料”列，展示文件标题和任务 ID；无来源显示“非导入来源”。
- 使用 Ant Design `rowSelection`。勾选后显示批量操作栏：已选数量、来源文件、统一备注输入框以及三个审核按钮。
- 若选择跨来源题目，操作栏显示阻断提示并禁用三个批量审核按钮。
- 从资料导入页进入时，保留当前来源资料卡与原文片段，并以标题而非孤立任务 ID 描述当前筛选。
- 成功后清空选择、刷新当前列表与工作台统计；失败走已有全局错误提示。

### 单题抽屉

- 抽屉顶部展示来源文件标题、任务 ID 与“查看原文”入口；无来源显示非导入来源。
- 将状态下拉框替换为三枚语义明确的按钮：通过、需修改、驳回；点击仅更新草稿，仍需点“保存审核”持久化。
- 保留题干、答案、标签、评分标准和单题发布能力。

## 错误处理与一致性

- API 错误统一通过现有全局 Ant Design 提示展示。
- 批量审核的跨来源、已发布或权限问题由服务端拒绝，前端不做乐观写入。
- 列表刷新前不清空用户未提交的单题编辑草稿；批量成功后才清空已选行。

## 验收与测试

- 合约测试：来源对象与批量审核输入的合法性、空值和状态边界。
- 服务测试：同源批量更新、跨来源拒绝、已发布候选题拒绝、审计记录。
- 前端测试：来源列、同源批量操作可用、跨来源操作禁用、状态按钮替代下拉框。
- 管理端 typecheck、lint、test、build；Product API typecheck、test。
