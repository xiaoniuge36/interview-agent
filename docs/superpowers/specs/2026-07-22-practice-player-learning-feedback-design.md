# 刷题工作台与学习反馈设计

## 目标

让用户能专注回答当前题目，并清楚知道 Agent 会在什么时机将练习结果写入能力画像、影响后续推荐。

## 已确认的学习机制

1. 单题回答保存只保留本轮练习内容。
2. 单题 AI 评价会生成评分和反馈，但不会立即写入 `masteryProfile`。
3. 用户生成整轮 AI 复盘后，系统按题目标签汇总评分、更新 `masteryProfile`；下一次推荐会优先考虑低掌握度标签并排除近期已练题目。
4. “结束自学（无 AI 报告）”不会更新能力画像。

## 设计方向

页面的对象是正在集中准备面试的求职者，单一任务是完成一题高质量回答。视觉采用“专注答题桌面”：低对比的冷白画布、一个宽阔的主答题卡、紧凑的横向题目进度轨道，以及固定在右侧的答题教练。

### 页面结构

```text
页面标题与总进度
────────────────────────────────────────────────
本轮题目 · 1 2 3 4 5       状态图例
───────────────────────┬────────────────────────
当前题目与答案编辑器   │ 答题教练
保存状态与上下题       │ 解析 / AI 评价
                       │ Agent 学习轨迹
───────────────────────┴────────────────────────
```

### 学习轨迹

在答题教练末尾增加 `PracticeLearningNotice`。

- 未完成单题评价：说明保存并完成 AI 评价后，会在整轮复盘时更新相关能力画像。
- 已完成单题评价：说明本题反馈已就绪；完成整轮 AI 复盘后，会把关联标签的证据写入能力画像并影响下一轮推荐。

此组件不会暗示“即时学习”，不增加 API、数据库或评分逻辑。

### 视觉系统

- 画布：雾蓝 `#f5f8fc`、主卡 `#ffffff`、墨蓝 `#14233a`、行动蓝 `#2e67d2`、学习信号绿 `#16806c`。
- 布局：桌面双栏，题目导航横跨两栏；小屏恢复单栏和横向滚动题号。
- 签名元素：学习轨迹是一张带细竖线的“证据卡”，它只承载真实的持久化承诺，避免装饰性指标。
- 可访问性：保留语义标题、按钮焦点态，并在 `prefers-reduced-motion` 下停止提示动效。

## 变更边界

- 只修改用户端刷题页面组件与样式，并新增对应静态渲染测试。
- 不修改评分、持久化、推荐算法、API、合同、数据库或依赖。
- 保留当前回答保存、查看解析、调用 AI 评价和完成整轮复盘的行为。

## 验证

1. 静态渲染测试覆盖“等待评价”和“本题反馈已就绪”两种学习轨迹文案。
2. 运行刷题相关 Vitest 测试、用户端 TypeScript 与 ESLint。
3. 构建用户端，确认样式导入和页面编译。
