# 用户端恢复页、登录页与顶部账号操作设计

## 目标

让用户端的认证恢复更轻、更可信；让登录页以表单为重心；把已登录后的主题、个人设置与退出登录收拢到顶部操作区。

## 设计

### 认证恢复页

移除三步大向导，保留品牌、当前恢复状态和一条细进度轨迹。它表达“正在恢复会话”，而不是伪造后台执行进度；登录完成后自动进入目标页面。

### 登录页

保留双栏结构以承载 OfferPilot 的品牌气质，但左栏只保留核心承诺和两条真实能力说明；去掉五步旅程。右栏将登录表单放大为视觉重心，收紧留白与多余描述。

### 顶部账号操作

现有全局搜索顶栏的右侧新增账号操作区：主题切换、个人设置、用户身份和退出登录。侧边栏只保留品牌和主导航。开发模拟身份不显示退出登录，维持现有权限语义。

## 变更边界

- 只修改用户端 React 展示组件、CSS 和对应测试。
- 不修改认证协议、会话存储、主题偏好存储、路由、API、合同或依赖。
- 保留移动端底部导航；窄屏时顶部账号操作收缩为图标与头像。

## 验证

1. 认证恢复页和顶部账号操作的静态渲染测试覆盖关键文案及开发模式差异。
2. 运行用户端认证、账号操作相关 Vitest 测试。
3. 运行用户端 TypeScript、ESLint、生产构建和格式检查。
