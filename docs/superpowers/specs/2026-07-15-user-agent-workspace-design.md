# C 端 Agent 工作台与 BYOK 设计

## 目标

将 OfferPilot 从按流程分散的训练页面重组为面向求职者的个人 Agent 工作台，并让用户可以一次配置、长期安全使用自己的模型 Provider 与 API Key，驱动真实的模拟面试对话。

## 已确认的产品决策

- 受众是个人求职者，不是招聘方或管理员。
- API Key 由用户自己配置，持久化到本产品数据库，必须加密保存，浏览器与任何查询响应均不可取得原文。
- 模型调用必须真实发生；首版支持 OpenAI、Anthropic、DeepSeek、通义千问及 OpenAI-compatible 自定义端点。
- 用户只需要配置一次；可测试连接、替换、停用、删除和选择默认连接。
- 首页、设置中心、个人档案遵循已确认的 OfferPilot 深色侧栏、暖白工作区、钴蓝主操作、薄荷绿状态信号的视觉方向。

## 信息架构

```text
工作台
├── 首页               下一项行动、训练记忆、进度路径
├── 我的 Agent         个人档案、目标职位、证据与 Agent 记忆
├── 面试工作室         真实模型驱动的对话与追问
├── 专项练习           现有练习能力
├── 复盘库             现有报告能力
└── 设置中心
    ├── AI 模型        Provider 连接、模型选择、连接测试、默认项
    ├── 账号与安全     账户概要与会话安全入口
    └── 通知           面试、复盘和练习提醒偏好
```

## 体验设计

### 首页

首页不再把每一步都做成等权卡片。首屏先显示 Agent 是否已就绪、用户最值得做的下一件事，以及一个可读的准备记忆时间线。右侧常驻 Agent Memory：目标职位、准备度、已记录证据和下一步建议。没有 API Key 时，下一项行动明确引导至 AI 模型设置；已连接且测试成功后，主按钮直接开始或继续模拟。

### 我的 Agent

个人信息编辑被解释为“更新 Agent 记忆”：目标职位、年限、技能和项目证据会直接说明如何影响下一次追问。用户可编辑自己的档案；已有的画像 API 继续服务数据事实源。右侧显示从档案推导出的优势、待练点及影响，保存动作统一命名为“保存并更新 Agent 记忆”。

### 设置中心与 AI 模型

AI 模型页的默认状态是空状态：说明密钥只加密保存在服务端并只用于用户的 Agent 任务，按钮为“添加模型连接”。新增/编辑采用受控表单：Provider、可选自定义 Base URL、模型名、API Key 和“设为默认”。保存时只发送一次原始密钥；读取时只返回 Provider、模型、脱敏尾号、状态、最近测试时间和默认标记。测试动作由后端用该凭证完成最小真实请求，成功后更新连接状态。

### 交互与状态

- 任何需真实模型的面试启动前都检查用户是否存在默认且已验证的连接；缺少时返回可操作的 `MODEL_CONNECTION_REQUIRED` 错误，前端链接到设置。
- Provider 错误、超时、余额不足或无效密钥以准确可修复的中文提示展示，永不回显密钥或上游完整错误体。
- 删除连接要求二次确认；删除默认项会在同一事务中改选其它已验证项，或让用户没有默认模型。
- 键盘可达、焦点可见，移动端将侧栏收起为抽屉，内容单栏显示。

## 安全与数据设计

`UserModelCredential` 为用户拥有的凭证记录，不复用现有面向管理员的 `ModelProfile`。每个记录包含 Provider、模型、可选 Base URL、密文、12 字节 IV、16 字节 GCM Tag、密钥版本、脱敏提示、验证状态和最近测试时间。

密钥使用 AES-256-GCM 加密。32 字节 base64 主密钥只来自 `CREDENTIAL_ENCRYPTION_KEY` 环境变量，密钥版本来自 `CREDENTIAL_ENCRYPTION_KEY_VERSION`；生产环境没有有效值即拒绝启动。加解密代码只在 Product API 进程内使用，原始密钥从不写日志、审计 metadata、错误、数据库查询结果或浏览器响应。Provider 调用在 Product API 服务器内完成，避免把用户密钥传入浏览器或 Agent Runtime HTTP 请求。

## 真实模型调用

Product API 新增用户模型客户端，保持现有面试命令、幂等、事件与报告持久化边界不变。用户的默认已验证连接通过受控凭证服务解析；Provider 适配器构建约束 JSON 的面试决定请求，并只解析 `{stage, content, shouldFinish}`。OpenAI、DeepSeek、通义千问和自定义端点走 OpenAI-compatible 适配器，Anthropic 走原生 Messages 适配器。外部响应必须经过既有 Zod 运行时响应 schema 验证；校验失败、网络失败和上游拒绝会记录不含密钥的运行遥测并返回稳定错误码。

当前 Python Agent Runtime 保留为确定性开发/回归 fallback，不接收用户密钥；生产环境由用户模型客户端负责真实生成。

## 验收标准

1. 用户可创建、读取（仅脱敏）、更新、测试、设默认、停用与删除自己的模型连接；其他用户/租户不可访问。
2. 数据库、日志、API 响应、前端状态和错误中均不含 API Key 原文。
3. 配置完成的用户启动或继续面试时，系统真实调用对应 Provider，且模型输出满足现有 `AgentRuntimeNextResponseSchema`。
4. 未配置、失效或停用连接时，用户看到准确的操作提示并能进入 AI 模型设置。
5. 首页、设置、档案在桌面与手机宽度下均符合确认的 Agent 工作台设计；主路径可点击并更新真实本地 UI 状态。
