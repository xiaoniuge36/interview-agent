# 刷题 Agent 闭环与智能复盘设计

## 目标

修复用户端刷题“无法生成整轮复盘”的阻塞，并将个人信息、最近有效 JD、历史掌握度、逐题评价和下一轮推荐串成可解释的训练闭环。功能继续集成在现有用户端和 Product API 中，不新增独立系统。

## 已确认的问题

当前前端仅在所有题目都有逐题 AI 评价时启用“生成 AI 复盘”按钮，后端 `PracticeCompletionService` 也要求全部 `evaluation` 存在。用户只保存答案、没有逐题评价时，按钮不可用；即使手动调用接口，也会收到 `PRACTICE_EVALUATIONS_INCOMPLETE`。这使“逐题评价”和“整轮复盘”错误地形成强耦合。

## 方案

采用现有链路上的整轮复盘编排：用户保存全部答案后可以发起整轮复盘。若存在未评价题目，前端先弹出二次确认，明确提示将自动调用 AI 评价并消耗模型额度；用户确认后，后端复用现有模型评价能力补齐未评价题目，再生成报告和掌握度记录。

不采用异步任务队列或新增数据库表：本轮题量上限为 10，现有请求、事务和模型调用边界足以支持同步编排；若未来需要长任务，再独立演进任务状态模型。

## 用户体验

### 选题页

- 保留手动选题入口和 Agent 推荐题单。
- 推荐卡展示题目数量、预计时长、推荐来源和可解释理由。
- 推荐依据按优先级组合：最近有效 JD、个人目标岗位、历史低分能力、最近一轮缺失点、近期未覆盖能力。
- 近期已练题目进入排除集合；若个性化条件无匹配，依次降级到岗位分类和公共通用题。

### 答题页

- 题目导航同时展示“未回答 / 已保存 / 已评价”状态。
- 逐题 AI 评价继续保留，但不再是整轮复盘的前置条件。
- “生成 AI 复盘”在答案全部保存后可点击；未评价题目数量展示在按钮附近。
- 未评价题目存在时，二次确认文案包含数量和模型额度提醒。
- 确认后展示“补齐题目评价—生成整轮复盘—更新能力记录”的过程状态。
- “结束自学”继续作为不调用 AI 的独立出口。
- 任何失败都保留答案和已有评价，提示可重试，不将会话错误地标记为完成。

### 复盘页

- 展示总分、摘要、表现亮点、待加强能力、下一步行动和逐题评价。
- 展示本轮写回的掌握度变化或能力标签。
- 复盘报告生成后刷新推荐，让下一轮训练可以直接承接本轮薄弱项。

## 后端数据流

```text
保存全部答案
  -> POST /practices/:id/submit
  -> 校验会话归属与答案完整性
  -> 若有缺失逐题评价：调用 PracticeModelEvaluator 补齐并持久化
  -> 在同一受控完成流程中生成 PracticeReport
  -> 更新 MasteryProfile
  -> 会话状态 report_ready
  -> 返回结构化 PracticeReport
```

已有评价必须复用，避免重复消耗；重复提交已生成报告的会话必须幂等返回同一报告。模型调用失败时不生成半成品报告，已有答案和已有评价保持可重试状态。

## 推荐编排

推荐服务继续复用现有 API 和契约，但从单个弱标签升级为上下文评分：

- JD：优先使用最近更新的 `JobIntent`，读取岗位目标和可用能力字段；没有 JD 时回退个人档案岗位。
- 掌握度：取低分标签，并保留分数用于排序，不只取第一个标签。
- 历史表现：从最近已完成练习中读取题目和评价关联的标签，提升近期低分且尚未覆盖能力的优先级。
- 去重：排除最近练习题；排除集合不足时允许按能力降级，不返回空推荐。
- 解释：返回组合后的推荐理由，至少说明岗位匹配、薄弱项或未覆盖原因之一。

本轮不引入新的 AI 推荐调用，先用可测试的确定性编排保证稳定和可解释；后续可在不改前端契约的情况下替换为 Agent Runtime 推荐器。

## 错误处理

- 未保存全部答案：返回现有 `PRACTICE_ANSWERS_INCOMPLETE`，前端提示仍需保存的题目。
- 未配置模型：返回现有 `MODEL_CONNECTION_REQUIRED`，前端保留“结束自学”出口并引导设置模型。
- 模型不可用或返回格式错误：返回现有模型错误码；报告不落库，用户可重试。
- 会话已关闭：保持现有幂等报告返回和 `PRACTICE_SESSION_CLOSED` 语义。
- 前端确认取消：不发送提交请求，不改变会话状态。

## 测试与验收

- 前端模型测试：全部答案但部分评价时，整轮复盘按钮可用；未保存答案时仍不可用。
- 前端交互测试：存在未评价题目时先确认，取消不调用 API，确认后进入提交状态。
- 后端单测：部分评价的会话会补齐未评价题目并生成报告；已有评价不重复调用模型；模型失败不生成报告。
- 后端并发测试：同一会话重复提交只生成一份报告并返回同一结果。
- 推荐单测：JD + 个人档案 + 多个掌握度弱项 + 最近练习会产生可解释、去重后的推荐；个性化无匹配时能降级。
- 运行 Product API 定向测试、用户端 typecheck/lint/build，并在本地服务启动后用浏览器验证选题、作答、确认复盘、报告展示和下一轮推荐。

## 范围边界

- 不新增系统、服务或独立后台。
- 不新增数据库表和迁移。
- 不在浏览器端拼接模型 prompt；模型调用继续留在 Product API。
- 不把“结束自学”改成 AI 行为。
