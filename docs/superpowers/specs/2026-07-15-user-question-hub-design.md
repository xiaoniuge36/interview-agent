# C 端自主刷题与 Agent 学习闭环设计

## 1. 目标

将登录后的 C 端首页从“准备度工作台”调整为可立即使用的刷题入口，同时保留 OfferPilot 的 Agent 差异化能力。用户可以自由浏览公共题库、筛选专题、勾选 1–10 道题并开始练习；档案、目标岗位和历史能力记录只影响 Agent 推荐，不成为自主刷题的前置门槛。

成功标准：

- 登录用户进入 `/home` 后能直接搜索或选择题库专题。
- 用户能在独立选题页筛选并勾选 1–10 道题。
- 用户能以单题沉浸模式完成作答、保存、查看解析和真实模型反馈。
- 修改档案或目标岗位后，下次请求自动更新 Agent 推荐，但不覆盖用户手动题单。
- AI 评价只调用用户已验证的默认模型连接，不使用本地规则冒充 AI。
- 完成本轮练习后，复盘和能力标签回写并影响下一次推荐。

## 2. 已确认的产品规则

1. 首页采用“题库大厅 + Agent 侧栏”。
2. 专题进入“独立选题页”，不在首页原地展开，也不使用桌面专属抽屉。
3. 作答采用“单题沉浸式作答”，一次聚焦一道题。
4. 没有个人档案或目标岗位仍可浏览和刷公共题。
5. 档案、岗位和 mastery 只增强 Agent 推荐。
6. 无模型连接仍可浏览、选题、保存回答和查看标准解析。
7. AI 评分、遗漏项和追问必须真实调用用户默认模型。
8. 模型失败时答案和进度必须保留，用户可重试、换连接或继续自学。

## 3. 用户业务流程

### 3.1 首页题库大厅

首页主区域依次提供：

- 搜索框：按题目标题、题干和可见标签搜索。
- 分类筛选：岗位方向、题型、难度和热门专题。
- 专题卡片：显示名称、题目数量、难度范围和代表标签。
- 最近练习：有未完成练习时提供“继续练习”。

右侧 Agent 推荐栏提供：

- 基于档案、岗位和 mastery 的推荐题单。
- 推荐原因，例如“目标岗位高权重能力”或“最近低分能力”。
- 一键采用推荐题单并创建练习。
- 无档案时显示通用精选题单，并提示完善档案可获得岗位化推荐。

题库大厅是主入口，Agent 推荐是辅助入口；推荐接口失败不影响用户浏览题库。

### 3.2 独立选题页

用户从专题卡片或搜索结果进入 `/questions`：

- 支持关键词、分类、标签、题型、难度和掌握状态筛选。
- 支持按推荐度、更新时间和难度排序。
- 题目列表只返回标题、题干摘要、题型、难度和标签，不返回参考答案与 rubric。
- 用户可勾选 1–10 题。
- 桌面端右侧固定显示已选题单；移动端使用底部汇总栏。
- 点击“开始练习”复用现有 `POST /practices`，以 `questionIds` 创建 manual session。

### 3.3 单题沉浸式作答

练习页改为三段结构：

- 左侧：题号、完成状态和整轮进度。
- 中间：当前题、题型/难度/标签、回答输入区、保存和导航。
- 右侧：标准解析、AI 评价、遗漏项、rubric 得分和针对性追问。

移动端使用顶部进度与底部操作栏，不保留固定侧栏。

答案保存与 AI 评价拆为两个独立动作：

1. 保存答案先落库，成功后更新题目状态。
2. 用户请求 AI 评价时再调用真实模型。
3. 模型调用失败不回滚已保存答案。
4. 修改已评价答案时，使旧评价失效，用户可重新评价。

### 3.4 整轮复盘

全部题目完成评价后生成本轮报告：

- 总体得分、强项、薄弱项和下一步动作。
- 每题评价和针对性追问。
- mastery 标签更新。
- 首页下一次推荐优先覆盖岗位高权重且当前低分的能力。
- 避免连续推荐本轮已完成题目。

没有模型连接时可以完成自学和查看标准解析。用户点击“结束本轮自学”后，session
进入 `submitted`，但不生成 AI 得分、AI 复盘和 mastery 分数；该动作不可再补做 AI 评价。

## 4. 前端信息架构

### 4.1 首页

- `QuestionHubPage`：首页数据编排与状态处理。
- `QuestionSearchBar`：搜索入口。
- `QuestionCategoryFilters`：分类、题型和难度快捷筛选。
- `QuestionTopicGrid`：专题卡片网格。
- `AgentRecommendationRail`：推荐题单、推荐原因和配置提示。
- `RecentPracticeCard`：恢复未完成练习。

### 4.2 独立选题页

- `QuestionPickerPage`：筛选、分页与选中状态编排。
- `QuestionFilterPanel`：桌面筛选栏与移动筛选弹层。
- `QuestionCatalogList`：题目列表与空状态。
- `SelectedQuestionTray`：已选 1–10 题和开始练习入口。

### 4.3 作答页

- `PracticePlayer`：当前题索引与导航。
- `PracticeQuestionNav`：题号和状态。
- `PracticeQuestionStage`：题目与回答编辑。
- `PracticeCoachPanel`：标准解析、真实模型评价和失败重试。
- `PracticeCompletionPanel`：整轮报告和下一步入口。

每个文件保持单一职责，新建文件不超过 300 行，函数不超过 50 行。

## 5. API 与数据契约

### 5.1 题库目录

新增 `GET /question-catalog`，参数包括：

- `query`
- `category`
- `tags`
- `type`
- `difficulty`
- `sort`
- `page`
- `pageSize`

响应包括题目摘要列表、分页信息和筛选聚合数量。查询范围只能包含 public 或当前 tenant 的 published 题目。

### 5.2 Agent 推荐

新增 `GET /practice-recommendations`：

- 有档案/岗位/mastery 时按岗位类别、高权重能力和低分标签组合推荐。
- 无档案时返回通用精选或最近更新题单。
- 每次请求读取最新数据，不维护需要失效的推荐缓存。
- 返回题目 ID、标题、推荐原因、预计时长和题目数量。

### 5.3 最近练习

新增 `GET /practices/recent`，返回当前用户最近一个未完成 session 的摘要，用于首页恢复入口。

### 5.4 答案保存与真实模型评价

保留 `POST /practices/:id/answers/:itemId` 作为答案保存接口。

新增 `POST /practices/:id/items/:itemId/evaluate`：

- 读取已保存答案、参考答案、rubric 和可见标签。
- 使用当前用户已验证的默认模型凭证。
- 通过 `ModelProviderClient` 调用真实 Provider。
- 要求模型返回结构化 JSON，并用 Zod 严格校验。
- 持久化 score、feedback、missingPoints、rubricScores 和 followUpQuestion。
- 返回评价与标准解析。
- 不存在默认模型时返回 `MODEL_CONNECTION_REQUIRED`。
- 鉴权、限流、超时、Provider 异常和 JSON 无效使用稳定错误码。

### 5.5 无模型自学完成

新增 `POST /practices/:id/complete-self-study`：

- 要求每道题都已保存答案。
- 将 session 更新为 `submitted`。
- 不创建 `PracticeReport`，不写入 mastery。
- 返回更新后的 `PracticeSession`。
- session 提交后禁止继续修改答案或请求 AI 评价。

## 6. 数据库变更

在 `EvaluationResult` 增加：

```prisma
followUpQuestion String?
```

该字段保存本题针对性追问，保证刷新和继续练习后仍可恢复。参考答案继续读取 `Question.answer`，不重复存储。

修改已评价答案时删除当前 item 的旧评价，并把状态恢复为 `answered`；重新评价后写入新结果。

## 7. 真实模型评价

新增 `PracticeModelEvaluator`，复用：

- `ModelCredentialService.resolveDefault()`
- `ModelProviderClient.complete()`

模型输入包含：题目、用户答案、参考答案、rubric、标签和目标岗位。模型必须返回：

```json
{
  "score": 0,
  "feedback": "具体反馈",
  "missingPoints": ["遗漏项"],
  "rubricScores": [{ "point": "评分点", "score": 0 }],
  "followUpQuestion": "针对当前回答的追问"
}
```

禁止使用 `PracticeEvaluator` 的本地规则结果作为 AI 反馈。旧规则评价器只可保留给非 AI 内部测试，不能进入新的 C 端评价路径和报告元数据。

## 8. 错误与空状态

- 题库为空：保留筛选条件并提供清空筛选。
- 推荐不可用：题库大厅继续可用，侧栏提供重试。
- 无档案：显示热门推荐和完善档案提示。
- 无模型：允许自学，AI 区域提供“连接模型”。
- 模型鉴权失败：提示测试或更新连接。
- 模型限流/服务异常：保留回答并提供重试。
- 模型 JSON 无效：提示本次反馈不可用，不保存脏数据。
- 题单不足 1 题或超过 10 题：前后端同时阻止创建。
- 题目失效或无权访问：创建练习时返回明确选择失效错误。
- session 已提交：禁止继续修改答案或评价。

## 9. 视觉方向

视觉基调继续使用 OfferPilot 的深海军蓝、亮蓝与薄荷绿，不复制参考站的黄色题库商城风格。

页面签名元素是“Agent 推荐理由”：每个推荐题单不仅显示题目数量，还显示来自岗位、档案或能力证据的理由，让用户知道推荐为什么与自己有关。

题库区域保持高信息密度，但只在专题卡片与筛选上使用圆角；作答区减少卡片嵌套，突出当前问题与回答。

## 10. 测试与验收

### 自动化测试

- 契约测试：目录查询、推荐、逐题评价和错误响应。
- API 测试：tenant/public 范围、筛选、分页、题单校验、所有权和最近练习。
- 推荐测试：无档案、岗位匹配、mastery 低分和档案更新后的结果。
- 模型评价测试：真实调用适配层、JSON 校验、无连接、鉴权、限流和无效响应。
- 持久化测试：答案保存、旧评价失效、评价重试和 followUpQuestion 恢复。
- 前端纯函数测试：筛选模型、已选题单上限、推荐展示和作答导航。

### 浏览器验收

- 桌面与 390px 移动端首页。
- 无档案用户可进入题库并手动选题。
- 选题 1–10 题限制与移动底部题单。
- 单题导航、草稿保存、刷新恢复和继续上次。
- 无模型提示、连接模型入口和真实模型评价结果。
- 模型失败后答案不丢失。
- 完成整轮后报告和首页推荐发生变化。

## 11. 范围边界

本轮包含题库目录、首页、独立选题、单题作答、真实模型逐题评价、最近练习和推荐闭环。

本轮不包含收藏夹、公开排行榜、会员计费、社区讨论、用户自建公开题库、题目评论和多人对战。
