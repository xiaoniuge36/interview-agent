# 用户端全局搜索设计

## 目标

将首页当前“提交后直接跳转题库页”的搜索表单，改造成类似 React 文档站的居中命令面板。用户可以先看到分组结果并用键盘浏览，只有明确选择结果后才导航。

## 范围

首版搜索以下三类公开信息：

- 已发布题目：通过现有 `GET /question-catalog?query=` 实时查询。
- 题库专题：使用用户端已有专题定义，在浏览器内匹配名称与描述。
- 功能页面：搜索首页、自主刷题、我的 Agent、面试工作室、练习空间、复盘中心和设置。

首版不搜索个人练习记录、面试记录、报告正文或模型连接，避免结果噪声和个人数据索引扩张。

## 交互

### 打开方式

- 首页搜索框点击、聚焦或输入时打开命令面板，并把当前输入同步到面板。
- 用户端任意页面支持 `Ctrl+K` / `Cmd+K` 打开。
- 首页搜索按钮不再提交跳页，而是打开面板。

### 面板结构

- 居中浮层，背景使用低透明遮罩和轻微模糊，视觉上沿用 OfferPilot 当前主题变量。
- 顶部为搜索输入框，右侧显示 `ESC` 提示。
- 空查询展示常用专题和功能入口。
- 有查询时按“题目、题库专题、功能页面”分组；题目最多展示 6 条。
- 每个题目结果展示标题、题型/难度和最多两个标签；专题与页面展示类型标识和简短说明。
- 底部展示键盘操作提示和结果数量。

### 键盘与关闭

- `ArrowDown` / `ArrowUp` 在可见结果间循环移动。
- `Enter` 打开当前结果。
- `Escape` 关闭并把焦点还给触发控件。
- 点击遮罩关闭；点击面板内部不关闭。
- 打开时禁止页面背景滚动。

### 结果行为

- 题目：导航到 `/questions?query=<题目标题>`，由题库页展示匹配题目。搜索过程本身不跳页。
- 专题：导航到 `/questions?category=<专题>`。
- 功能页面：导航到对应用户端路由。

## 数据流

1. `GlobalSearchProvider` 挂载在 `UserShell`，统一管理打开状态、查询词和触发元素。
2. 首页 `QuestionSearchBar` 调用 Provider，不再直接操作 Router。
3. `GlobalSearchDialog` 对查询词做短延迟处理，并调用已有 `getQuestionCatalog`。
4. 静态专题和页面入口通过纯函数在本地过滤。
5. 所有结果被转换为统一 `GlobalSearchItem`，供鼠标和键盘共享选择逻辑。
6. 导航完成后关闭面板并清理临时状态。

## 状态与错误

- 空查询：展示推荐入口，不请求题库 API。
- 加载中：保留已有静态结果，并显示轻量加载状态。
- API 失败：题目组显示可重试提示，专题和页面搜索继续工作。
- 无结果：明确提示“没有找到”，并提供“进入完整题库”入口。
- 使用请求序号忽略过期响应，避免快速输入时旧结果覆盖新结果。

## 响应式与可访问性

- 桌面面板最大宽度约 680px，顶部留出浏览上下文。
- 小于 640px 时变为带安全边距的近全屏面板，结果区域内部滚动。
- 使用 `role="dialog"`、`aria-modal`、`combobox`、`listbox`、`option` 和 `aria-activedescendant`。
- 保留清晰焦点环，并遵守现有 `data-motion="off"` 微动效设置。

## 测试

- 纯函数测试：静态入口匹配、结果合并顺序、活动索引移动。
- 组件行为：打开/关闭、空态、加载态、错误态、键盘导航和结果选择。
- 回归：首页搜索不再直接 `router.push`；题库页原有筛选与分页不变。
- 浏览器验证：桌面与 375px 移动端、控制台无错误、无横向溢出。

## 非目标

- 不新增数据库表、搜索引擎或共享 Contract。
- 不索引用户隐私数据。
- 不在首版实现搜索历史、拼音搜索、模糊纠错或高亮算法。

## 补充：全站常驻入口

- `GlobalSearchTrigger` 作为公共组件挂载在 `UserShell` 内容区顶部，所有登录后的用户页面共享。
- 顶部入口使用 `position: sticky` 固定在内容滚动区上沿，不遮挡侧栏与移动端底部导航。
- 桌面端展示“Agent 导航”、搜索文案、`Ctrl K` 和“题目 · 专题 · 页面”范围；窄屏逐步收敛为“搜索全部”。
- 点击入口仍调用同一个 `GlobalSearchProvider`，不复制查询、结果渲染或导航逻辑。
