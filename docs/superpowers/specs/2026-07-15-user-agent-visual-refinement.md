# C 端 Agent 控制台视觉修订

## 目标

以用户提供的 OfferPilot 参考图为唯一视觉基准，重构用户端首页和设置中心的呈现方式；保留已经落地的账号、模型连接、加密存储和真实模型调用行为。

## 视觉令牌

- 深色导航：`#071525`，用于 260px 固定左侧栏。
- 画布：`#FBFBFA`，用细灰分割线和低圆角区分信息层级。
- 主行动：`#155EEF`，仅用于开始模拟、保存和当前导航。
- 正向状态：`#12B981`，用于 Agent 在线、连接正常和已完成步骤。
- 表达方式：正文使用现有系统中文字体；标题采用紧凑字距和更高字重，避免渐变背景、玻璃拟态和无业务含义的装饰。

## 页面结构

### 全局侧栏

品牌、当前账户、主导航和底部个人设置按参考图的垂直节奏排列。主导航只显示用户真正使用的六个入口：首页、我的 Agent、面试工作室、专项练习、复盘中心和设置；目标岗位仍从“我的 Agent”工作流进入，不作为独立侧栏项。

### 首页

首页顶部是横向的 Agent 在线状态与“开始模拟”按钮。主工作区采用 `minmax(0, 1fr) / 420px` 双栏：左栏为大标题、继续准备卡和四步准备记忆时间线；右栏是白底、常驻的 Agent memory，显示目标岗位、真实准备度、证据和推荐行动。数据为空时，仍显示可执行的下一步，不展示虚构的个人信息或完成度。

### 设置中心

保留 AI 模型、账户与安全、通知三个 Tab。AI 模型 Tab 的头部包含说明和新增连接按钮；每个连接以一张完整的详情卡展示 Provider、连接状态、模型名、脱敏密钥和最后测试时间。连接编辑仍使用原有受控表单与 API，API Key 只在本次写入时发送。详情卡下方固定显示加密说明，并提供默认模型、回答风格、数据使用许可等模型行为偏好控件。

## 交互与响应式

- 所有原有连接的创建、编辑、测试和删除行为不变，按钮名称与结果提示保持一致。
- 低于 1180px 时首页记忆卡移到主区下方；低于 820px 时侧栏折叠成顶部品牌栏、首页单列、设置详情卡堆叠。
- 键盘焦点、禁用状态、错误提示和 `prefers-reduced-motion` 行为保持可访问。
