# 平台运营脉搏设计

## 目标

将现有后台的数据看板从区间计数展示提升为“平台经营健康”视图；同时把账号管理从宽松的筛选表格，优化为可快速识别、筛选和处置账号风险的治理工作台。

## 范围

- 保持功能在现有治理控制台内：`#accounts` 与 `#analytics`。
- 扩展既有 `GET /admin/platform/dashboard`，提供同一查询区间内的按日趋势、内容漏斗和运营提醒。
- 不新增数据库表、不迁移数据、不引入第三方图表库。
- 使用 Ant Design、原生 SVG 和现有 CSS token 完成图表与样式。

## 账号治理工作台

1. 页面首屏加入四个紧凑摘要：账号总量、窗口新增、窗口活跃、禁用账号。
2. 搜索保持常驻；租户、类型、角色、状态、来源移入可展开的高级筛选区。
3. 表格账号列渲染身份块：首字母头像、显示名、邮箱或主体；角色、来源、状态渲染为可扫描的语义标签。
4. 账号操作保留详情、改角色、启用/停用；在窄屏或高密度场景收束到“更多”菜单，避免操作列撑宽。
5. 摘要数字来自已加载的账号查询结果，并明确标记为“当前筛选结果”或“当前页”，避免伪造全局统计。

## 平台运营脉搏看板

页面按“总览、趋势、经营链路、运行风险”四个层级排列：

1. **健康总览**：显示运营健康状态、账号总量、窗口新增、训练完成数、Agent 成功率和平均延迟；健康状态由成功率、Schema 通过率、失败/降级数计算，并展示计算依据。
2. **经营趋势**：展示所选今日、7 天或 30 天窗口的按日新增账号、内容发布、训练完成、Agent 调用。使用零依赖 SVG 折线/柱状图，鼠标与键盘可读取每个时间点的数值。
3. **内容与训练链路**：以真实数据展示导入、待审核、发布题目、训练提交、训练报告；呈现发布率、报告完成率和待审核积压提示。
4. **运行风险**：展示 Agent 成功率、Schema 通过率、平均延迟、降级次数，并列出近期失败/降级运行和可操作的处理建议。

## 数据接口

`PlatformDashboard` 在保留现有字段的基础上新增：

- `trend`: 按日期升序的桶，字段为 `date`、`accountsCreated`、`questionsPublished`、`trainingCompleted`、`agentRuns`。
- `funnel`: `imports`、`pendingCandidates`、`publishedQuestions`、`practiceSubmissions`、`practiceReports`，所有字段都在同一统计窗口或明确标注为当前存量。
- `alerts`: 待审核积压、失败导入、运行失败/降级各自的数量与严重级别。

后端使用现有 Prisma 实体按 UTC 日分桶；空日期补零。前端只渲染接口返回的事实数据，百分比由明确的分子/分母计算，零分母显示为 `0%`。

## 错误、权限与响应式

- 看板仅平台管理员可见，保持现有 `analytics:read` 平台范围策略。
- 查询失败、无权限、无数据继续复用全局传统提示与居中加载状态。
- 桌面端双列排布；小于 992px 时趋势和漏斗单列，账号筛选与操作不横向溢出。
- 不将缺失数据展示为成功或 0：字段缺失应触发现有接口校验错误；仅真实空桶补 0。

## 验收

- 账号页在 1440px 与 768px 下均可完成筛选、重置、导出、详情、角色调整和启停入口访问。
- 看板在 today、7d、30d 切换后，趋势点数量与时间窗口一致，且与 API 返回的数值逐项一致。
- 运营漏斗、风险提醒和近期失败列表能正确处理零数据、异常数据与权限拒绝。
- 产品 API、contracts、admin-console 的定向测试通过；后台 TypeScript、Lint 与渲染冒烟检查通过。
