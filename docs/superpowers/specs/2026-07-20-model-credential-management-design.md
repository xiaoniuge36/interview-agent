# 模型连接管理设计

## 目标

在现有后台 Agent 助手的设置入口中管理当前账号创建的模型连接：新增、编辑、测试、启用或停用、删除。功能只扩展当前后台，不新增独立模型系统或菜单页面。

## 范围

- 右上角设置入口打开“模型连接”管理弹窗。
- 弹窗展示当前账号拥有的连接：服务商、模型、端点、状态、默认标记、密钥掩码、最近测试时间。
- 支持新增、编辑、测试、设为默认、停用、启用和删除。
- 编辑时不回显 API Key。留空表示保留现有密钥；填写新值表示轮换密钥。
- 删除前展示影响说明。删除默认连接后由后端自动提升另一条已验证连接；历史 AI 调用保留但解除凭证关联。

## 安全边界

- API Key 仅在新增或轮换时从密码输入框提交给 Product API，不写入浏览器存储、Agent 对话、审计 metadata 或接口响应。
- Product API 使用 AES-256-GCM 保存 `ciphertext`、`iv`、`authTag` 和 `keyVersion`；每次加密使用新的随机 IV。
- 前端只展示后端返回的 `keyHint`，不请求或重建明文密钥。
- 修改模型、端点或 API Key 后状态重置为“待测试”，只有测试成功后才能被选作默认连接。
- 所有写操作继续由现有权限策略与审计日志保护。

## 交互设计

1. 现有助手设置按钮打开模型连接管理弹窗。
2. 弹窗顶部提供“新增模型连接”。列表行提供“编辑”“测试”“设为默认”“停用/启用”“删除”。
3. 编辑连接使用与新增一致的表单；API Key 标签明确标记为“留空则不修改”。
4. 删除使用二次确认，展示模型名和默认连接切换说明。
5. 保存、测试、状态变更与删除成功后刷新列表及当前 Agent 配置；失败用现有全局错误提示显示后端原因。

## 数据流

前端复用现有 `GET/POST/PATCH/DELETE /model-credentials` 和 `POST /model-credentials/:id/test`。不新增 Prisma schema、迁移或后端写路径。现有 Product API 负责鉴权、加密、默认连接切换、历史引用解绑与审计。

## 验收标准

- 添加的模型可在同一后台入口编辑、测试、启停和删除。
- API Key 在列表、编辑回显、网络返回内容和审计 metadata 中均不出现明文。
- 轮换 API Key 后连接必须重新测试；删除默认连接后可用的已验证连接成为默认。
- 管理端 lint、typecheck、测试与 build 通过；Product API 模型凭证定向测试通过。
