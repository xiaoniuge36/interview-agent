请解释 React Server Components 与 Client Components 的边界。

React Server Components (RSC) 和 Client Components 的边界主要体现在执行环境和对客户端资源的依赖上。RSC 只在服务端渲染，不包含交互逻辑（如 useState, useEffect）且不增加客户端 bundle size。它们适合用于数据获取和静态内容渲染。

Client Components 则在客户端执行，可以处理用户交互和状态管理。边界通常通过在文件顶部使用 'use client' 指令来划分。当 Server Component 导入一个带有 'use client' 的模块时，这就形成了一个序列化边界，传递给 Client Component 的 props 必须是可序列化的。
